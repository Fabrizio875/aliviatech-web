# Source Code Style Guide & Conventions (Yerico)

- **Language**: English for all identifiers (files, classes, CSS classes).
- **HTML**: semantic elements (`header`, `nav`, `section`, `main`, `footer`). Accessible labels/alt, `aria-` attributes when needed.
- **CSS**: BEM-ish simple classes. Palette from report (primary teal #14B8C4, gradient blue #3B82F6â†’#8B5CF6, accent #A78BFA). Use `:root` variables.
- **JS**: Vanilla, modular when possible. Keep functions small. No inline scripts.
- **Accessibility**: Focus states, sufficient contrast, alt text, labels, `aria-live` for form status.
- **Performance**: Use system fonts fallback, limit heavy assets. SVG over large raster images.
